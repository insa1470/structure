## 討論日期
2026-04-30

## 一、痛點與目標

同仁目前工作流：

```
企查查截圖（圖一：股權結構圖 + 圖二：集團概覽）
  → 眼睛看圖
  → 手動打字進 Excel / Word
  → 調整格式、畫圖
  → 改版時重做一次
```

目標：用工具取代手工錄入，讓機器先做 80%，人只確認 20%。

---

## 二、技術選型決策

### 選擇：Claude Vision（claude-sonnet-4-6）

評估過的選項：

| 方案 | 優點 | 缺點 | 結論 |
|------|------|------|------|
| Google Cloud Vision | 傳統 OCR 成熟 | 只抓文字，不理解關係 | 不適合 |
| PaddleOCR | 免費、本地、中文強 | 需自己寫結構解析邏輯 | 備選（資安需求時） |
| Qwen-VL-Max | 中文最佳、費用低 | 需另申請阿里雲帳號 | 備選 |
| **Claude Vision** | 視覺推理最強、整合最簡單 | 費用略高於 Qwen-VL | **採用** |

### 選擇理由

- 視覺推理能力最強，直接理解節點連線關係與層級結構
- 與現有開發環境整合最簡單（同一個 Anthropic API Key）
- 每次任務成本約 $0.02，可忽略不計

### Claude Vision 能力評估

| 能力 | 評估 |
|------|------|
| 公司名稱識別（含生僻字） | 強 |
| 持股比例讀取 | 強 |
| 上下層關係推斷 | 強（標準樹狀圖 80–90% 準確） |
| 交叉持股 / 多層複雜結構 | 容易出錯，需人工確認 |

不確定的項目加 `"uncertain": true`，直接送進審核台的待確認清單。

## 三、輸入圖片說明

- **圖一**：企查查股權結構圖（樹狀節點 + 連線 + 持股比例）
- **圖二**：集團公司 / 子公司概覽（表格式，含法人代表、資本額、成立時間等）
- 兩張圖資料互補，需整合成一份主表

## 四、實作計畫

### 階段一：接入 Qwen-VL（目前優先）

目標：把 `POST /api/tasks/analyze` 從示範資料種子換成真實 API 分析。

**工作項目：**

- [ ] 新增 `analyzer.py`，封裝 Qwen-VL API 呼叫邏輯
- [ ] 實作 `analyze_chart1(image_path)` — 解析股權結構圖，回傳節點與關係
- [ ] 實作 `analyze_chart2(image_path)` — 解析集團概覽，回傳公司屬性
- [ ] 實作 `merge_charts(chart1_result, chart2_result)` — 整合兩張圖資料
- [ ] 在 `server.py` 的 `handle_task_analyze` 中呼叫 analyzer

**Qwen-VL Prompt 設計（圖一）：**

```
這是一張企查查股權結構圖。
請識別所有公司節點和持股關係，輸出以下 JSON 格式：

[
  {
    "company": "子公司名稱",
    "parent": "上層公司名稱",
    "shareholding_ratio": "51.2%",
    "level": 2,
    "uncertain": false
  }
]

如果某個關係不確定，請將該項目的 uncertain 設為 true。
只輸出 JSON，不要其他說明文字。
```

**Qwen-VL Prompt 設計（圖二）：**

```
這是一張企查查集團公司概覽圖或表格。
請提取每家公司的基本資訊，輸出以下 JSON 格式：

[
  {
    "company": "公司名稱",
    "legal_representative": "法人代表",
    "registered_capital": "1000萬元",
    "established_date": "2015-03-01",
    "company_type": "有限責任公司",
    "uncertain": false
  }
]

只輸出 JSON，不要其他說明文字。
```
